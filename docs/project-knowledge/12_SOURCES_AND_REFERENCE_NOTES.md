# Sources and Reference Notes

Use official documentation when implementing provider and cloud details. Confirm current service availability, quotas, credentials and region compatibility before provisioning.

## Azure
- Azure Container Apps with GitHub Actions: https://learn.microsoft.com/en-us/azure/container-apps/github-actions
- Azure Container Apps managed identity: https://learn.microsoft.com/en-us/azure/container-apps/managed-identity
- Azure OpenAI / Foundry keyless connections: https://learn.microsoft.com/en-us/azure/developer/ai/keyless-connections
- pgvector on Azure Database for PostgreSQL: https://learn.microsoft.com/en-us/azure/postgresql/extensions/how-to-use-pgvector
- Azure Static Web Apps build configuration: https://learn.microsoft.com/en-us/azure/static-web-apps/build-configuration
- Azure for Students: https://azure.microsoft.com/en-us/free/students

## Use of references
Do not bulk copy external source content into the repository. The RAG knowledge base should contain short, original, attributed security notes created for SARP.
